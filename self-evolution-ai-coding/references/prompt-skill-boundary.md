# Skill、Prompt、Memory、Evaluator 边界

## 一句话结论

Skill 管稳定流程，Prompt 管当次任务，Memory 管长期状态，Evaluator 管验收证据。

全局编码 Skill 只沉淀多项目通用的工程规则，不放项目私有角色、业务设定、内容生产流程或产品世界观。

## Skill 负责什么

Skill 适合放：

- 稳定工作流。
- 工程约束。
- 验收规则。
- 工具调用方法。
- 可复用检查清单。
- 多项目都适用的失败模式和预防规则。

Skill 不适合放：

- 一次性需求。
- 当前用户的临时偏好。
- 大段项目私有上下文。
- 产品角色设定或业务世界观。
- 每次都会变化的生成内容。
- 全量历史日志。

## Prompt 负责什么

Prompt 适合放：

- 本次任务目标。
- 输出格式。
- 当前素材。
- 当前约束。
- 当次场景的具体生成策略。

Prompt 不适合放：

- 长期工程规范。
- 所有历史记忆。
- 所有 Skill 内容。
- 复杂业务规则的唯一来源。
- 公开 AI 入口的长期运行逻辑。

旧 Prompt 可以作为 Skill seed、Policy seed 或兼容数据，但不能继续作为公开运行入口。

## Memory 负责什么

Memory 适合放：

- 长期稳定偏好。
- 已验证事实。
- 重要决策原因。
- 未来会影响行为的事件。
- 多次任务后稳定出现的用户或项目约束。

Memory 不适合放：

- 可从数据库重新查询的数据。
- 临时草稿。
- 可从 Skill 得到的通用方法。
- 无决策价值的聊天全文。
- 敏感密钥、真实凭据或不可脱敏的隐私数据。

## Evaluator 负责什么

Evaluator 适合做：

- 产物格式检查。
- 功能验收。
- 安全边界检查。
- 回归对比。
- LLM 输出质量评估。
- Trace、日志、测试和人工验收证据汇总。

Evaluator 不负责生成核心业务内容，也不替代测试、日志、数据库核验或人工审批。

## AI/Agent 入口边界

涉及模型调用、Agent 运行时、工具调用或模型池时，必须明确：

- 唯一 owner。
- 输入输出契约。
- 可用工具和权限。
- Prompt 来源和版本。
- Memory 读取范围。
- Evaluator 和验收信号。
- Trace、计费、错误处理和回滚策略。

禁止把旧 Prompt、旧 AIService、临时脚本或前端拼接逻辑继续当公开运行入口。

## 按需加载策略

1. 先识别任务类型。
2. 只读取对应 Skill 和 reference。
3. 只召回影响当前决策的 Memory。
4. 只把本次必要事实放入 Prompt。
5. 输出后判断是否需要写入错题本或全局自进化规则。

禁止一次性加载所有 Skill、所有 Memory、所有项目文档。
