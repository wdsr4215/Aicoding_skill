# Skill、提示词、记忆和场景专家的边界

## 一句话结论

skill 管稳定能力，prompt 管当次生成，记忆管长期状态，场景专家管专业判断。

## Skill 负责什么

skill 适合放：

- 稳定工作流。
- 工程约束。
- 验收规则。
- 工具调用方法。
- 可复用检查清单。
- 多次项目都适用的经验。

skill 不适合放：

- 一次性需求。
- 当前用户的临时偏好。
- 大段项目私有上下文。
- 每次都会变化的创作内容。
- 全量历史日志。

## Prompt 负责什么

prompt 适合放：

- 本次任务目标。
- 输出格式。
- 角色口吻。
- 当前素材。
- 当前约束。
- 当次场景的具体生成策略。

prompt 不适合放：

- 长期工程规范。
- 所有历史记忆。
- 所有 skill 内容。
- 复杂业务规则的唯一来源。

## 记忆负责什么

记忆适合放：

- 长期稳定偏好。
- 用户/角色/关系/世界状态。
- 已验证事实。
- 重要决策原因。
- 未来会影响行为的事件。

记忆不适合放：

- 可从数据库重新查询的数据。
- 临时草稿。
- 可从 skill 得到的通用方法。
- 无决策价值的聊天全文。

## 场景专家负责什么

场景专家是数字员工的一种。

它负责：

- 判断场景目标。
- 选择专业方法。
- 提出候选方案。
- 解释取舍。
- 记录错题。

例如“灵感扩展系统”应演进为“灵感专家”：

```text
灵感专家
  -> 场景：虚拟创作 / 商业短片 / 文旅内容
  -> 记忆：用户偏好、历史采纳方向、热点效果
  -> skill：灵感扩展方法、热点引用规则、题材安全检查
  -> prompt：本次灵感、目标受众、输出格式
  -> 反思：用户采纳/拒绝原因，沉淀下次建议
```

## 按需加载策略

1. 先识别任务类型。
2. 只读取对应 skill 和 reference。
3. 只召回影响当前决策的记忆。
4. 只把本次必要事实放入 prompt。
5. 输出后判断是否需要写入记忆或错题本。

禁止“一次性加载所有 skill、所有记忆、所有项目文档”。
