# 数字员工与数字生命底座

## 核心观点

数字员工不是一段固定提示词，而是“角色状态 + 记忆 + 技能 + 工具 + 权限 + 任务闭环”的运行单元。

所有数字员工共享同一套运行时架构，但拥有不同内容：

```text
通用运行时
  -> 意图路由
  -> 场景路由
  -> 记忆路由
  -> skill 路由
  -> 模型路由
  -> 工具执行
  -> 反思
  -> 记忆写入

数字员工实例
  -> 角色身份
  -> 专业领域
  -> 关系记忆
  -> 任务权限
  -> 领域知识
  -> 提示词模板
  -> 错题本
```

## 数字员工类型示例

- 灵感专家：负责随机灵感、热点引用、灵感扩展、题材建议。
- 大纲专家：负责结构、节奏、冲突、人物成长。
- 商业短片导演：负责品牌目标、镜头、转化路径、成片节奏。
- 文旅导览员：负责地点知识、游客画像、路线、讲解口吻。
- 代码审查员：负责风险、测试、耦合、维护性。

## 自我进化

数字员工的进化不等于自己改系统代码。更合理的进化层级：

1. 会话内调整：根据当前反馈修正输出。
2. 任务后反思：记录哪里判断错、哪里需要补充。
3. 记忆更新：保存稳定偏好、事实、关系和世界状态。
4. skill 提升：把可复用流程沉淀成规则。
5. 工具扩展：在人工批准后新增工具、脚本或服务。

## 架构边界

- 运行时负责调度，不负责具体专业内容。
- 数字员工负责专业判断，不直接拥有基础设施。
- 场景服务负责输入、流程和产物契约。
- 记忆服务负责存取、压缩和召回。
- skill 负责稳定方法和工具路由。
- prompt 负责当次任务的具体生成指令。

## 按需加载

数字员工启动时不加载全部记忆和全部 skill。

推荐流程：

```text
用户输入
  -> 判断场景
  -> 判断数字员工角色
  -> 读取当前任务最小必要记忆
  -> 读取对应 skill/reference
  -> 组装本次 prompt
  -> 执行
  -> 反思是否写入记忆或错题本
```
