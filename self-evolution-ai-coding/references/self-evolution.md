# 自我进化与错题本机制

## 目标

AI coding 必须从长期项目实践中吸收经验，避免同类错误反复发生。

## 反思触发条件

出现以下情况时必须反思：

- 构建、测试、运行时错误。
- 需求理解偏差。
- 文件边界混乱。
- 新旧实现重复。
- 临时文件污染正式项目。
- 用户指出“不好维护”“不够聚焦”“太复杂”“又变成屎山”。
- 发现之前的规则没有覆盖当前风险。

## 反思流程

```text
问题发生
  -> 定位触发信号
  -> 判断根因
  -> 给出修复
  -> 加入预防扫描
  -> 记录到 AI_CODING_MISTAKE_NOTEBOOK.md
  -> 如果是稳定通用规则，吸收到全局 skill 或 reference
```

## 错题本条目格式

```text
### 编号. 问题标题

触发信号：
- ...

根因：
- ...

预防：
- ...

扫描命令：
```text
...
```

修复原则：
- ...
```

## 吸收规则

- 项目私有问题进入项目错题本。
- 多项目通用问题进入 `self-evolution-ai-coding` reference。
- 长规则进入 reference，不进入 `SKILL.md`。
- 只有稳定、可复用、可验收的经验才上升为通用规则。

## 不做的事

- 不把每次临时错误都写成全局规则。
- 不在 skill 中记录项目私有细节。
- 不把日志当作知识库无限堆积。
