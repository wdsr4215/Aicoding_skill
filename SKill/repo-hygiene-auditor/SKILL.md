# repo-hygiene-auditor

## 1. 目标

用于清理仓库、根目录、废弃文件、孤立模块和生成产物。目标是在不误删用户数据、不破坏构建、不丢历史的前提下，让项目保持干净可维护。

## 2. 触发条件

- 用户要求清理根目录、删除无用文件、整理项目。
- 发现临时导出、旧压缩包、日志、数据库、缓存、构建产物混入源码。
- 需要判断文件是否仍被引用。
- 需要生成 `.gitignore` 或调整忽略规则。
- 需要同步仓库前做依赖检查。

## 3. 核心原则

1. 先依赖检查，再删除。
2. 先分类，再行动。
3. 运行数据、上传文件、数据库、密钥默认不提交。
4. 大量删除、清空目录、覆盖配置必须进入 `safety-gate`。
5. 不删除用户未确认的业务资料。
6. 清理后要更新地图和部署账本的相关坑点。

## 4. 文件分类

| 分类 | 示例 | 默认处理 |
|---|---|---|
| 源码 | `src/`, `app/`, `api/`, `services/` | 保留 |
| 配置 | `package.json`, `vite.config.ts`, `.env.example` | 保留 |
| 敏感配置 | `.env`, secret, key | 不提交，必要时打敏感包 |
| 运行数据 | `uploads/`, `*.db`, logs | 不提交，不随意删除 |
| 构建产物 | `dist/`, `.next/`, `build/` | 忽略，可清理 |
| 缓存依赖 | `node_modules/`, `__pycache__/` | 忽略，可清理 |
| 交付包 | `handoff/*.zip` | 按需保留，默认不进仓库 |
| 临时脚本 | `tmp_*`, `check_*`, `dump_*` | 先查引用，再决定 |

## 5. 工作流

### Step 1：列清单

使用 `rg --files`、`git status --ignored`、目录列表识别候选文件。

### Step 2：依赖检查

对候选文件执行：

- 文件名引用搜索。
- import/require 引用搜索。
- 脚本入口检查。
- 文档/部署账本引用检查。
- package scripts 检查。

### Step 3：行动策略

- 明确无用且低风险：删除或加入忽略。
- 可能有历史价值：移动到归档目录。
- 敏感但团队需要：进入 handoff sensitive 包。
- 不确定：保留并记录原因。

### Step 4：验证

根据项目执行最小验证：

- 构建。
- 单测。
- 类型检查。
- 核心入口 smoke。

## 6. 禁止行为

- 未检查引用直接删除。
- 删除 `uploads/`、数据库、生产备份。
- 把 `.env` 加入 Git。
- 用 `git clean -fd` 这类命令绕过审查。
- 把清理报告写成流水账。

## 7. 联动规则

必须更新 `TOPOLOGY.md`：

- 删除、移动、重命名核心目录或入口。
- 新增归档策略或 Known Pitfall。

必须更新 `DEPLOY_MANIFEST.md`：

- 清理影响部署脚本、构建入口、服务器路径、运行目录。
